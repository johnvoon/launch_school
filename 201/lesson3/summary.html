Inline Elements

inline elements always fit to content and are unaffected by width and height values.

inline elements accept margin and padding but only push other elements horizontally away, not vertically.

Block Elements

A number of elements are set to block by the browser user agent stylesheet, e.g. container elements like div, section and ul and text elements like p and h1.

Default display values can be overridden.

Provided no width is set, block elements take up the width of the page or parent container.

They respond to width, height, margin and padding. height will adjust to fit content if not set.

If width is set, you can set margin: 0 auto to horizontally centre the element within the container.

If max-width is set, element width will automatically adjust as windows is re-sized.

width: 0; height: 0
width: 0; height: 60px; padding: 20px; margin: 20px.
width: 90px; height: 0.

Inline-Block Elements

inline-block elements stay inline with the natural flow of text, but you can consistently apply block element properties to them, such as width, height, margins, padding and vertical-align while remaining an inline element. They behave similarly to a float in that several inline-block elements will line up horizontally. inline-block elements are therefore incredibly useful when you want to align block elements horizontally in rows.

For example, in blogs, we frequently use unordered lists for the navigation menu. Turning that list horizontal would require you to float all menu items left or right, then deal with the height of the containing element using a clearfix, etc. All of these problems are resolved by applying display: inline-block to all the li elements, which are block level elements by default. As shown in the above example with the anchor elements, you can make them fill up the space by applying display: block to them.

The advantage of inline-block elements is you don't have to clear them to stop the parent from collapsing around them.

Also, you can use vertical-align: middle, line-height and text-align: center to perfectly center the elements, in an intuitive way.

Float

Only position: static and position: relative elements will accept float properties.

A floated element is taken out of normal flow then shifted left or right.

A floated element is automatically display: block. Elements which are display: block by default remain so when floated. Therefore, floated inline elements will respond to margin, padding, height and width values.

The advantage of using floats is that when images are resized, surrounding text will reflow to accommodate it. Not the case for position: relative and position: absolute though

float's sister property is clear. To prevent elements after a floated element in the HTML wrapping around floated elements, we apply clear: both to the div following the floated elements so that it moves itself down past the float.

Clearing left or right is less common but nevertheless has its specific use cases:

The great collapse

If a parent element contains only floated elements, its height collapses. We fix it by clearing the float after the floated elements in the container

If you always know what the succeeding element will be, you can apply clear: both to it and go about your business. If not, there are other tools and hacks:

  • The Empty Div Method: Sometimes you will see a br element but an empty div is most common because it has no browser default styling, no special function, and is unlikely to be generically styled with CSS. But scorned by semantic purists as it adds no contextual meaning to the page and is purely for presentation. But it gets the job done.
  • The Overflow Method: Set overflow on parent container to auto or hidden and it will expand to contain the floats; succeeding elements will clear them.

    This makes the parent establishe a block formatting context and "take responsibility" for its content.

    Floats, absolutely positioned elements, block containers that are not block boxes (e.g. inline-block), and block boxes with overflow other than visible establish new block formatting contexts for their contents.

    This method can be beautifully semantic but be careful not to accidentally hide content or trigger unwanted scrollbars!

    As overflow: hidden creates a new block formatting context, this is useful to align a block element next to a floated one rather than allow its content to wrap around the floated element.

    In a block formatting context, boxes are laid out one after the other, vertically, beginning at the top of a containing block. The vertical distance between them is determined by margin. Vertical margins between adjacent block boxes in a block formatting context collapse. In a new block formatting context, each box's leftmost edge touches the leftmost edge of the containing block (for left-to-right formatting). This is true even in the presence of floats (although line boxes may shrink due to floats), unless the box establishes a new block formatting context, in which case the box would align next to the float and its width would shrink. See this stackoverflow answer for more details.
  • The Clearfix Method: Adds a clever CSS pseudo element after the parent element which clears the floats. 
    .clearfix:after {
  content: " ";
  display: block;
  clear: both;
  }

Positioning

Floats are of limited use as they can only position elements left and right. There are 4 types of positioning:

  • Static
  • Fixed
  • Relative
  • Absolute

Containing blocks of different elements

The position property manipulates the location of an element.

Positions or dimensions of elements are relative to the containing block.

The dimensions of the initial containing block, being the root element, is same as the viewport.

  • static/relative: Content box of nearest ancestor whose display is block or inline-block;
  • absolute: Padding box of nearest ancestor whose position is absolute, fixed or relative;
  • fixed: Viewport

Static

Default position for all elements, meaning no position setting. Element remains in normal document flow. left, right, top, bottom and z-index have NO effect.

Fixed

Element removed from normal flow of document and positioned relative to the viewport, unaffected by scrolling. top, right, bottom and left properties are used to position fixed elements.

The blue box on the bottom left of the page is a fixed position element.

Relative

No immediate effect on positioning. But activates top, bottom, left or right to move the element relative to its place in the document. Remains in normal document flow and surrounding elements will remain unmoved.

Setting relative position also activates z-index on that element. Even if no z-index value is set, this element will be placed on top of static positioned elements.

Absolute

Removes element from document's normal flow. Thus, the space it once occupied will be taken up by other elements.

absolute position elements are positioned relative to their closest relative position ancestor. If such element does not exist, they will be positioned relative to the body element.

Take note that position: absolute inline and inline-block elements become block elements and therefore respond to width and height values.
